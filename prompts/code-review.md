## 你的任务

从以下维度对代码进行严格审查：

1. **功能实现**
   - 代码是否符合技术方案的设计
   - 是否完整实现了需求的验收标准
   - 是否有遗漏或超出范围的改动

2. **代码质量**
   - 可读性和代码结构是否良好
   - 命名规范是否与现有代码保持一致
   - 是否有明显的代码重复或改进空间

3. **潜在问题**
   - 边界情况和异常处理是否完善
   - 并发问题、竞态条件
   - SQL 注入、越权等安全隐患
   - 是否有内存泄漏或资源泄露风险

4. **性能**
   - 是否有明显的性能隐患
   - 数据库查询是否有 N+1 问题
   - 新增的索引是否足够支撑查询

5. **测试**
   - 是否有相关的单元测试或集成测试
   - 测试是否通过
   - 测试覆盖是否足够

6. **前端视觉**（仅涉及前端改动）
   - 视觉效果是否符合设计方案
   - UI 交互是否符合用户预期
   - 是否有响应式设计问题或兼容性问题

## 审查标准

- **高严重程度**：会导致功能不正常、数据丢失或安全问题
- **中严重程度**：影响性能、代码可维护性或用户体验
- **低严重程度**：代码风格、非关键的改进建议

**通过的条件**：无高严重程度问题，中等问题不超过 3 个且解决方案清晰。
